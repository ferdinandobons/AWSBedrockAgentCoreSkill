# Project memory — aws-bedrock-agentcore-skill

Claude Code plugin bundling the `aws-bedrock-agentcore-skill` skill (`skills/aws-bedrock-agentcore-skill/`):
a source-cited playbook for building AI agents on AWS (Strands Agents, Amazon Bedrock, Bedrock
AgentCore). See `README.md`. Repo: https://github.com/ferdinandobons/AWSBedrockAgentCoreSkill (public).

## Maintenance & freshness — IMPORTANT

This skill documents a **fast-moving** domain (Bedrock AgentCore went GA in late 2025; features ship
monthly). It has a real shelf life. Treat it as **self-correcting and well-verified, not guaranteed
bug-free** — every QA pass so far found real runtime bugs in freshly-added code snippets.

- **After ~3–6 months, treat maturity labels, version pins, GA dates, and every "Verify live" item as
  SUSPECT** until re-checked against the official page in `references/sources.md`.
- **Refresh process**: periodically re-run the build/QA workflows — research → build-simulation audit →
  full per-file review → **snippet verification against official docs**. The workflow scripts used to
  build and verify this skill live locally in `.aws-research/` (gitignored build artifacts; re-runnable
  or regenerable). The source index is generated by `python3 .aws-research/gen_sources.py`.
- **Bump the version** in `.claude-plugin/plugin.json` on every shipped change; run
  `claude plugin validate .` before committing.
- **Never hard-code volatile facts** (live prices, default quotas, current model IDs/regions) — defer to
  the AWS console / model cards, as the skill already does.

## Authoring rules (when adding or editing content)

- Every best practice and code snippet **must carry an inline `_Source:` official URL**.
- **New code snippets must be verified against official docs piece by piece** (params, shapes/types,
  enums, ARNs, imports) — do not trust generated code; the QA passes repeatedly caught
  wrong-param / list-vs-dict / str-vs-bytes / deprecated-API bugs.
- **Provenance: official sources only** — `docs.aws.amazon.com`, `strandsagents.com`, official AWS
  GitHub orgs (`aws`/`awslabs`/`aws-samples`/`aws-ia`), the official Terraform registry
  (`hashicorp/aws`, `hashicorp/awscc`), and `boto3`/`pypi`/`npm`. The two non-AWS sources (the A2A
  protocol spec and Langfuse as an optional OTEL backend) are labelled as such; third-party blogs are
  excluded.
- Keep `SKILL.md` a router (decision tree + playbooks); put depth in `references/`. On any conflict,
  the `references/` file wins over the `assets/` matrices and over the `SKILL.md` summary.
- After adding sources, regenerate `references/sources.md` and re-run the link check before committing.
