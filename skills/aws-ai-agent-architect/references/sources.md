## Source policy

Every recommendation in this skill is traceable to a source below (327 unique official URLs). Provenance tiers:

- **AWS service docs** — `docs.aws.amazon.com` (Bedrock, AgentCore, IAM, CloudWatch, Prescriptive Guidance) and `aws.amazon.com/blogs` (official AWS blog).
- **AWS open-source SDK** — `strandsagents.com` and the `github.com/strands-agents`, `github.com/aws`, `github.com/awslabs`, `github.com/aws-ia` orgs (official / AWS-maintained). `boto3.amazonaws.com` and `pypi.org` for the AWS SDK / package index.
- **Terraform provider (canonical, HashiCorp-official)** — `registry.terraform.io` (`hashicorp/aws`, `hashicorp/awscc`) and `hashicorp.com`. These are the authoritative reference for the Terraform AWS providers, distinct from AWS service docs.
- **Cross-vendor standard (not AWS)** — `github.com/a2aproject/A2A` is the official Agent-to-Agent protocol spec that Strands implements. Treat as the canonical protocol source, but it is **not** an AWS service.
- **Optional third-party** — `langfuse.com` appears only as an *optional* OTEL observability backend. The AWS-native path (CloudWatch / AgentCore Observability) is always the primary recommendation; Langfuse is a documented alternative, not required.
- **Excluded** — third-party blogs (e.g. `awsfundamentals.com`) are **not** cited; where such a source informed a fact, the canonical AWS / provider source is cited instead.
